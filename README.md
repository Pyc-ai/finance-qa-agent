# 金融知识问答 Agent(RAG + 工具调用)

基于大语言模型的金融领域智能问答 Agent:通过 **RAG(检索增强生成)** 回答金融专业问题并标注资料来源,通过 **Function Calling(工具调用)** 完成利息等精确计算,采用 **ReAct 循环** 实现多步推理。

> 项目背景:作者曾在银行信用卡中心从事信贷业务,将业务知识整理为知识库,构建了这个面向金融场景的问答智能体。

## 架构

```
用户提问
   │
   ▼
┌─────────────── Agent 主循环 (agent.py) ───────────────┐
│                                                        │
│   大模型思考 ──→ 需要工具? ──否──→ 输出最终答案         │
│       ▲              │是                               │
│       │              ▼                                 │
│       │        执行工具 (tools.py)                     │
│       │       ┌──────────────────────┐                 │
│       └───────│ · 知识库检索 (rag.py) │                │
│    结果回填    │ · 计算器              │                │
│               │ · 当前日期            │                 │
│               └──────────────────────┘                 │
└────────────────────────────────────────────────────────┘

RAG 流程: 文档 → 切块 → 向量化 → 余弦相似度检索 → 注入 prompt
```

## 快速开始

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置 API Key(阿里云百炼,有免费额度: https://bailian.console.aliyun.com)
export DASHSCOPE_API_KEY=sk-你的key     # Windows 用 set 命令

# 3. 运行
python main.py
```

示例问题:`贷款30万,年利率4.5%,每月利息多少?根据知识库,提前还款有什么限制?`

Agent 会自动:① 调用计算器算出月利息 → ② 检索知识库获取提前还款规则 → ③ 综合两者生成带来源的回答。

## 文件说明

| 文件 | 作用 |
|---|---|
| `agent.py` | ReAct 主循环:思考 → 调工具 → 看结果 → 再思考 |
| `tools.py` | 工具定义(JSON Schema)与执行:检索 / 计算器 / 日期 |
| `rag.py` | RAG 模块:文档切块、向量化、余弦相似度检索 |
| `config.py` | 模型与参数配置 |
| `knowledge_base/` | 知识库文档(Markdown),可自由替换扩充 |

## 技术要点

- **Function Calling**:模型不执行代码,只输出"调用意图"(JSON),由程序解析执行后将结果以 `role=tool` 回填上下文
- **RAG**:embedding 向量化 + 余弦相似度检索,让模型基于私有知识回答并标注来源,抑制幻觉
- **上下文管理**:大模型无状态,每轮请求携带完整对话历史与工具结果
- **安全**:计算器使用 `ast` 白名单解析而非 `eval()`;API Key 走环境变量,不入库
